Thank you for installing {{ .Chart.Name }}!

1. To see the service and port:

  kubectl get svc -l app={{ include "ollama.name" . }} -n <namespace>

2. Port-forward to access the Ollama server locally:

  kubectl port-forward deploy/{{ include "ollama.fullname" . }} 11434:{{ .Values.service.port }} -n <namespace>

Open http://localhost:11434 (or the port you forwarded).

3. To change the model or storage size at install/upgrade time:

  helm upgrade --install my-release ./ollama/hello-pvc \
    --namespace my-ns --create-namespace \
    --set model.name=mistral7b \
    --set pvc.size=20Gi

  4. Ingress

    If `ingress.enabled` is true the chart creates an Ingress. By default it
    uses host `ollama.local` and `ingressClassName: nginx`. Add a hosts entry
    to `/etc/hosts` for testing:

      127.0.0.1 ollama.local

    Then access http://ollama.local

6. GPU / NVIDIA notes

    If you plan to run Ollama on nodes with NVIDIA GPUs, follow these prerequisites:

      - NVIDIA drivers must be installed on the node(s).
      - Deploy the NVIDIA Device Plugin or the NVIDIA GPU Operator so the resource
        `nvidia.com/gpu` is available to the scheduler.

    To enable GPU via the chart values, set `gpu.enabled=true` (defaults to false) or use `-f gpu-values.yaml`.

    Example: enable GPU at install time

      helm upgrade --install ollama ./ollama -f ./ollama/gpu-values.yaml --set gpu.enabled=true

    After deployment check pod allocation and resources:

      kubectl describe pod <ollama-pod>  # check Allocated resources contains nvidia.com/gpu: 1
      kubectl logs <gpu-plugin-pod> -n kube-system  # (if troubleshooting)
